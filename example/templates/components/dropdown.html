Dropdowns display contextual menus. Use the text attribute for the button label and variant for button color. Inside, use c-dropdown.item for menu items, c-dropdown.header for section headers, and c-dropdown.divider for separators. {% cotton:verbatim %} {% endcotton:verbatim %} The dropdown uses c-button internally, so all button attributes are supported. Pass any attribute like variant, size, text, icon, etc. to customize the dropdown button. See the Button component documentation for full customization options. {% cotton:verbatim %} {% endcotton:verbatim %} By default, dropdowns display a caret icon to indicate they are expandable. Set :caret="False" to hide the caret and create a cleaner button appearance. {% cotton:verbatim %} {% endcotton:verbatim %} Use the direction attribute to control where the menu opens. Values: "down" (default), "up", "start", "end". {% cotton:verbatim %} {% endcotton:verbatim %} Control the minimum width of the dropdown menu using the min-width attribute. Default is "10rem". Accepts any valid CSS width value. Note: Dropdown menus will always grow to fit their content regardless of this setting. This attribute is most helpful when using non-list dropdown menus (see below) where you want to ensure a minimum width for form inputs or other custom content. {% cotton:verbatim %} {% endcotton:verbatim %} By default, dropdowns render menu items within a <ul> element. For custom content like forms, text, or other non-list items, set :ul="False" to render the menu as a <div> instead. This is particularly useful when combined with min-width to control form widths. {% cotton:verbatim %}
